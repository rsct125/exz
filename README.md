# 🎓 دليل تشغيل ونشر بوابة استعلام اختبارات المتدربين على Netlify

تمت إعادة هيكلة وتطوير المشروع بنجاح لفصل واجهة البحث عن قاعدة البيانات، بهدف **حماية بيانات المتدربين وسريتها**، وتسريع تحميل الصفحة وتوفير واجهة تفاعلية زجاجية (Glassmorphism) فائقة الجودة.

---

## 🏗️ الهيكل الجديد للمشروع
*   `public/`: المجلد العام المتاح للزوار (يحتوي على واجهة البحث الخفيفة والسريعة).
*   `netlify/functions/`: يحتوي على الكود البرمجي الآمن الذي يُشغل محرك البحث على خوادم Netlify دون إظهار قاعدة البيانات للجمهور.
*   `netlify/functions/exz.json`: ملف قاعدة البيانات الآمن المخزن على الخادم.
*   `netlify.toml`: ملف إعدادات إعادة التوجيه والتشغيل الخاص بـ Netlify.

---

## 💻 التشغيل والتجربة محلياً (Local Development)

إذا كنت ترغب في تشغيل واختبار المشروع على جهازك قبل رفعه، اتبع الخطوات التالية:

1.  **تثبيت بيئة Node.js:** تأكد من تثبيت Node.js على جهازك.
2.  **تثبيت Netlify CLI:** افتح منفذ الأوامر (Terminal) في مجلد المشروع، وقم بتثبيت أداة Netlify بالطلب التالي:
    ```bash
    npm install -g netlify-cli
    ```
3.  **بدء التشغيل محلياً:** شغل المشروع عبر الأمر التالي:
    ```bash
    netlify dev
    ```
    سيقوم هذا الأمر بتشغيل سيرفر محلي ومحاكاة الدوال البرمجية (Serverless Functions) تلقائياً، ويمكنك فتح الرابط المعروض (غالباً `http://localhost:8888`) لتجربة واجهة البحث والـ API.

## 🚀 النشر على Netlify (Deployment)

يمكنك نشر هذا المشروع على Netlify مجاناً بإحدى الطرق التالية:

### الطريقة الأولى: الرفع المباشر عبر واجهة Netlify (سهل جداً وبدون كود)
1.  قم بضغط مجلد المشروع بالكامل في ملف واحد بصيغة `.zip`.
2.  ادخل إلى حسابك في موقع [Netlify](https://www.netlify.com/).
3.  انتقل إلى تبويب **Sites** واسحب ملف الـ `.zip` وضعه في منطقة الرفع المباشر **(Drag and drop your site folder here)** أسفل الصفحة.
4.  سيقوم Netlify بقراءة ملف `netlify.toml` تلقائياً، وسيقوم بفصل مجلد النشر ومجلد الدوال البرمجية ونشر موقعك خلال ثوانٍ معدودة.

### الطريقة الثانية: النشر المرتبط بـ GitHub (الموصى به للتحديث التلقائي)
1.  قم بإنشاء مستودع (Repository) جديد على حسابك في GitHub.
2.  ارفع ملفات المشروع إلى المستودع.
3.  ادخل إلى موقع Netlify واضغط على **Add new site** ثم اختر **Import an existing project**.
4.  اختر **GitHub** وقم بتفويض Netlify للوصول إلى المستودع الخاص بك.
5.  اختر مستودع هذا المشروع، ودع الإعدادات الافتراضية كما هي (سيقوم بقراءة الإعدادات تلقائياً من `netlify.toml`).
6.  اضغط على **Deploy site**. الآن، في كل مرة تقوم فيها بتحديث ملف البيانات `exz.json` ودفعه إلى GitHub، سيقوم الموقع بالتحديث تلقائياً!

## 🔒 كيف تم تأمين البيانات؟
في النسخة السابقة، كانت كامل قاعدة البيانات (التي تحتوي على أسماء جميع المتدربين وأرقامهم الأكاديمية ومقاعدهم وقاعاتهم) مدمجة داخل كود الصفحة البرمجي، مما يعني أن أي شخص يستطيع فتح أدوات المطورين بالمتصفح وتحميل ملف البيانات بالكامل.

**في الهيكل الجديد:**
- لا يستطيع المتصفح الوصول نهائياً لملف `exz.json`.
- عندما يقوم المستخدم بالبحث، ترسل الصفحة طلباً مشفراً إلى الرابط السري `/api/search?id=رقم_الطالب`.
- تقوم الدالة البرمجية (المخفية على خوادم Netlify) بالبحث داخل الملف وإرجاع نتيجة الطالب المحدد فقط دون غيره.
